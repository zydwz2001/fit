# Vibe Fitness Pro

一个面向个人训练复盘的健身记录产品原型。

它把训练记录、身体指标、照片对比和知识沉淀放进同一个工作台里，重点解决个人健身场景里最真实的碎片化问题：训练时不好记、训练后不好复盘、变化数据不好串起来。

## 可交互 Demo

- 发布入口：`docs/index.html`
- 本地预览：直接用浏览器打开 `docs/index.html`
- GitHub Pages：当前 workflow 会直接发布 `docs/` 目录，不依赖本地构建

Demo 覆盖基础功能：

- 训练：今日训练、组数完成切换、加组、添加动作、总容量计算
- 回顾：月视图、训练日标记、日期详情
- 身体：体重 / 腰围 / 臂围 / BMI 曲线、新增记录、本地保存
- 知识库：文件夹展开收起、新增笔记、新建文件夹、本地保存

## 项目背景

真实的个人健身流程里，记录动作和组数只是第一步，更大的问题是后续如何把训练量、身体变化和知识经验串成连续复盘。传统工具往往只能覆盖其中一段，导致训练、身体数据和笔记彼此割裂。

## 技术方案

- 前端采用 Vite + React + TypeScript，适合快速搭建原型和持续迭代。
- 用 React Context 和 reducer 管理训练、身体和知识三条数据链路。
- 使用 `localStorage` 做本地优先存储，降低原型验证成本。
- 通过 SVG 和 Canvas 处理图表与照片对比，让身体变化可视化。

## 实现逻辑

- 训练页支持动作、组数、重量和左右侧数据的快速录入，并在录入时同步计算容量。
- 月视图把训练记录按日期聚合，用于观察部位分布和训练量趋势。
- 身体页把体重、BMI、围度等指标与日期绑定，形成可回看的变化轨迹。
- 照片对比能力把多张身体图放在同一画布中，方便做阶段对照。
- 知识模块用 Markdown 和文件夹结构沉淀动作经验和训练反思，形成“训练后复盘”的闭环。

## 设计取舍

- 优先服务训练时的输入效率，而不是先把界面做得很复杂。
- 优先验证“记录 - 复盘 - 沉淀”闭环，而不是一开始追求完整社区或社交能力。
- 以本地数据为中心，适合做个人产品原型和方向验证。
